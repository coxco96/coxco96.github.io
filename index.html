North American Breweries

This static map is one of my earliest mapping projects. It uses a hexagonal grid to show the number of breweries per 50 square kilometers throughout North America. In this project, I learned how to use design elements on static maps within QGIS.

Kentucky Opioids: The Sackler Family

This "scrollytelling" style web page features two maps presented in a journalistic style. I used R/RStudio to process a very large data file to make these interactive web maps. I made the maps using Leaflet with Javascript, and I created the "scrollytelling" experience using scrollama.js.

Patterns of U.S. Energy Production

This interactive map shows each hydro, wind, natural gas, and solar energy plant in the U.S. I created popup tooltips to show more information about each plant upon hover (or tap on mobile). I used circles proportionate to capacity in order to visually show energy source density. I incorporated design elements that allowed for a visualization of a lot of data without feeling cluttered. This map also includes interactive zoom capability.

U.S. Rent Prices by County

This interactive map shows average rent price by county in the U.S. I used the following packages in Javascript to create this map: Leaflet, jQuery, and Simple Statistics. With this map I demonstrate using statistical ckmeans breaks in order to cluster data for optimized visualization.

What the Sackler Family Owes Me (And My Fellow Kentuckians)

This project was created in December 2021 as the final project for a course (MAP 673: Interactive Design for Web Mapping).

Inspired by work in The Pudding and the New York Times, I wanted to explore a scrollytelling library and create a narrative to pair with the maps I had prepared.

I used scrollama.js for the scrollytelling element. The maps were made with Leaflet.js using data from the Washington Post's ARCOS API.

I learned how to work in R/RStudio to use the API.

Traffic Lights in Austin, TX

This map was created with Leaflet cluster groups. The color palette comes from Chroma, and the color styles are coded with visual coordination in mind.

Data for this project was cleaned using Mapshaper and GDAL/ogr on the command line, along with various Node.js packages.

The basemap was created with OpenStreetTiles and CARTO Voyager tiles. The data is sourced from Austin's Open Data Portal.

Reimagining remote sensing data visuals

In creating the "Where is it hottest in Miami-Dade County?" map, I ran into several challenges and unknowns. While vector temperature data exists for county averages, I needed to use remote satellite data to make this county map on the more granular level.

Remote sensing data is anything but pretty. It's traditionally difficult to map raster data due to the large file sizes of geoTIFFs.

For this map, I learned how to use Google Earth Engine to pull in remote sensing data. I used GEE to calculate the average land surface temperature of Miami-Dade County based on each day of summer 2021. I exported the data as a geoTIFF, which I brought into QGIS, where I trimmed the data precision, aggregated similar values, and recasted my raster data as vector data. From here, I exported a geojson file and brought that file into Flourish.

I chose the color scheme carefully for this map. I did not want to create an alarmist graphic, but I wanted to use colors that were both accessible and could accurately depict the reality of the situation.
